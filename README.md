# Online-Pharmacy-Portal
### Objective
The objective of this project is to develop an online shopping cart application for an online pharmacy. The application will cater to the sale of pharmaceutical products, offering both a user-friendly customer interface and a robust administrative interface. The project will prioritize server-side functionality, leveraging PHP with MySQL for data persistence.
#### Project Scope
•	Customer Interface:
 	Display a full list of products with details including unique ID, name, description, price, and stock quantity.
 	Implement search functionality allowing customers to find products by name or description.
 	Enable customers to select products and specify the desired quantity for purchase.
 	Upon order submission, the application will update the available product quantities, ensuring that back orders are not permitted.
 	Display a list of all previous orders placed by the customer.
•	Admin Interface:
 	Allow administrators to add, update, or delete products.
 	Provide a detailed list of all customers, with the ability to view individual customer order histories.
 	Enable administrators to update or delete existing customer orders.
•	REST APIs:
 	Provide RESTful APIs in both XML and JSON formats for:
	Retrieving the full product list.
	Retrieving products matching a specific name.
	Retrieving products within a specified price range.
### Technical Requirements
•	Backend Development:
 	Node.js with MySQL
	Utilize Express.js for handling server-side routing and logic.
	Use MySQL for relational data storage, ensuring robust handling of transactions and data integrity.
•	Frontend Development:
 	Develop a responsive and intuitive interface using HTML, CSS, and JavaScript frameworks (e.g., React, Vue.js).
 	Ensure cross-browser compatibility and mobile responsiveness.
•	APIs:
 	Implement RESTful APIs using Express (Node.js).
 	Ensure APIs are secure and provide data in both XML and JSON formats.
### Features and Functionality
•	Customer Interface Features:
 	Product catalog with search and filter options.
 	Shopping cart functionality with real-time stock updates.
 	Order history and tracking.
•	Admin Interface Features:
 	Product management (add, update, delete).
 	Customer management, including order history.
 	Order management with options to update or delete orders.
•	APIs:
 	Product listing and search.
 	Filtering products by name and price range.
 	Support for XML and JSON formats.


The proposed online pharmacy shopping cart application will provide a seamless shopping experience for customers while offering powerful tools for administrators to manage products and orders. With secure and scalable technologies, the project aims to deliver a reliable and user-friendly platform.
